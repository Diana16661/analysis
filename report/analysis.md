# Аналіз платформ: Android vs iOS vs Web

У цьому звіті проведено порівняльний аналіз трьох платформ:

* Android
* iOS
* Web (браузер)

Мета — оцінити архітектуру, доступ до ресурсів, безпеку та можливості розробки.

---

## Загальна характеристика платформ

### Android

* Тип: мобільна платформа
* Призначення: споживчі пристрої
* Типи пристроїв: смартфони, планшети, Smart TV

### iOS

* Тип: мобільна платформа
* Призначення: преміум-сегмент
* Типи пристроїв: iPhone, iPad

### Web

* Тип: web-платформа
* Призначення: універсальні застосунки
* Типи пристроїв: ПК, смартфони, планшети

---

## Архітектура та середовище виконання

### Android

* ОС: Android (на базі Linux)
* Ядро: Linux kernel
* Архітектура: ARM, x86
* Runtime: ART (Android Runtime)

### iOS

* ОС: iOS
* Ядро: Darwin (XNU)
* Архітектура: ARM
* Runtime: Native (Swift, Objective-C)

### Web

* ОС: не залежить від платформи
* Runtime: JavaScript engine (V8, SpiderMonkey)

| Ресурс          | Android       | iOS           | Web            |
| --------------- | ------------- | ------------- | -------------- |
| Камера          | Повний доступ | Через дозволи | Обмежений      |
| Сенсори         | Так           | Так           | Обмежено       |

| Мережа          | Повний доступ | Повний доступ | Повний доступ  |
| Фонові процеси  | Так           | Обмежено      | Обмежено       |

## Модель безпеки

### Android

* Система дозволів
* Ізоляція додатків (sandbox)
* Google Play Protect

### iOS

* Строгий sandbox
* Контроль через App Store
* Обов'язкові дозволи

### Web

* Same-Origin Policy
* HTTPS
* Ізоляція через браузер

### Android

* Material Design

### iOS

* Human Interface Guidelines
* Стандартизований UX

### Web

* Responsive design
* Залежність від браузера

## Переваги та обмеження

### Android

**Переваги:**

* відкритість платформи
* широкий доступ до ресурсів

**Обмеження:**

* фрагментація пристроїв

### iOS

**Переваги:**

* висока продуктивність
* стабільність

**Обмеження:**

* закритість екосистеми

### Web

**Переваги:**

* універсальність
* швидка розробка

**Обмеження:**

* обмежений доступ до апаратних ресурсів

## Висновки

* Найбільш гнучка платформа: Web
* Найкраща продуктивність: iOS
* Найшвидша розробка: Web
* Найкраща для крос-платформи: Flutter

### Доцільність використання:

* Android → масовий ринок
* iOS → преміум-застосунки
* Web → MVP та веб-сервіси
